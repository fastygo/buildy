# Buyer Persona: Agency Principal

## Profile

| Field | Detail |
|-------|--------|
| Title | Founder, Delivery Director, Practice Lead at digital agency |
| Company | 3–30 person studio building client web products |
| Reports to | Partners or ownership group |
| Tools today | Figma, proposal decks, mixed CMS (WP/Webflow), freelance dev network |
| BuildY role | Economic buyer; sets delivery methodology |

## Goals

- **Win more deals** with interactive previews during sales.
- Increase **margin per project** via templates and AI-assisted codegen.
- Deliver **maintainable** client handoffs (repo + CMS admin + docs).
- Differentiate from "cheap AI landing" competitors.

## Pain points

- Pitch prototypes don't match final deliverable stack.
- Clients trapped in agency retainer for trivial content edits.
- Junior devs inconsistent on AppCMS/FastyGo patterns.
- AI tools help sales but hurt delivery standardization.

## BuildY value proposition

> "Standardize client delivery on CMS-backed FastyGo apps: prototype in BuildY, export to client GitHub, train on AppCMS admin — faster pitches, cleaner handoffs."

## Decision criteria

1. Reusable templates per vertical  
2. White-label or client-branded preview story  
3. Export quality passable for agency QA  
4. Training time for PMs/designers on workspace  
5. Support model for AppCMS runtime post-handoff  

## Triggers to buy

- New FastyGo/AppCMS practice line  
- Lost deal because competitor showed working product faster  
- Support ticket volume from content-locked legacy deliverables  

## Package offerings enabled

| Package | BuildY role |
|---------|-------------|
| SaaS launch landing | Template + CMS blocks + export |
| Marketing site refresh | Content migration mapping |
| MVP app shell | Landing + admin preview + eng handoff |
